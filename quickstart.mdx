This guide will help you get Droguería Somos Salud running as quickly as possible. For detailed installation instructions, see the [Installation Guide](/installation).

## Prerequisites

<Note>
Before starting, ensure you have:
- PHP 7.0+ installed
- MySQL 5.7.14+ installed
- Apache 2.4+ with `mod_rewrite` enabled
</Note>

## 5-Minute Setup

<Steps>
  <Step title="Set Up Database">
    Create the database and import the schema:
    
    <CodeGroup>
    ```bash Terminal
    # Create database
    mysql -u root -p -e "CREATE DATABASE drogueria CHARACTER SET utf8 COLLATE utf8_swedish_ci;"
    
    # Import schema
    mysql -u root -p drogueria < db/database.sql
    ```
    
    ```sql SQL Console
    -- In MySQL console
    CREATE DATABASE drogueria CHARACTER SET utf8 COLLATE utf8_swedish_ci;
    USE drogueria;
    SOURCE db/database.sql;
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure Database Connection">
    Edit `db/db.php` with your MySQL credentials:
    
    ```php db/db.php
    $_user = 'root';        // Your MySQL username
    $_pass = 'your_pass';   // Your MySQL password
    ```
  </Step>

  <Step title="Enable mod_rewrite">
    Ensure Apache's mod_rewrite is enabled:
    
    ```bash
    sudo a2enmod rewrite
    sudo systemctl restart apache2
    ```
  </Step>

  <Step title="Access the Application">
    Navigate to the login page:
    
    ```
    http://localhost/login
    ```
    
    <Info>
    **Default Credentials:**
    - Username: `admin`
    - Password: `123456`
    </Info>
  </Step>
</Steps>

## First Login

After logging in with the default credentials, you'll see the main dashboard. The system uses role-based authentication defined in `index.php:5-7`:

```php
if (!isset($_SESSION['rol']) || $_SESSION['rol'] == "" || $_SESSION['rol'] == "Sin registrar") {
    $_SESSION['rol'] = "Sin registrar";
    header('location:/login');
}
```

<Warning>
**Important:** Change the default admin password immediately after your first login for security!
</Warning>

## Essential First Steps

<Steps>
  <Step title="Change Admin Password">
    1. Navigate to **Users** in the sidebar
    2. Click on the admin user
    3. Edit and update the password
    4. Save changes
  </Step>

  <Step title="Add Your First Laboratory">
    Products require a laboratory (manufacturer). Add one first:
    
    1. Go to **Laboratories** → **Create**
    2. Enter the laboratory name (e.g., "Bayer", "Pfizer")
    3. Save
  </Step>

  <Step title="Add Product Presentations">
    Define how products are presented:
    
    1. Go to **Presentations** → **Create**
    2. Add presentation types (e.g., "Tablets", "Capsules", "Syrup")
    3. Save
  </Step>

  <Step title="Create Your First Product">
    Now you can add products to inventory:
    
    1. Navigate to **Articles** → **Create**
    2. Fill in the product details:
       - Name
       - Laboratory (from dropdown)
       - Presentation (from dropdown)
       - Stock quantity
       - Minimum stock (for alerts)
       - Sale price
    3. Save the product
    
    <Note>
    The product schema is defined in `db/database.sql:20-33` with fields for inventory tracking and pricing.
    </Note>
  </Step>

  <Step title="Add a Supplier">
    To make purchases, you need suppliers:
    
    1. Go to **Suppliers** → **Create**
    2. Enter supplier information:
       - Name
       - NIT (Tax ID)
       - Address
       - Phone
       - Email
    3. Save
  </Step>
</Steps>

## Core Features Overview

### Inventory Management

The system tracks inventory with automatic stock updates:

- **Articles:** Manage products with stock levels and minimum alerts
- **Laboratories:** Organize products by manufacturer
- **Presentations:** Define product formats

### Purchase Management

- **Suppliers:** Maintain supplier database
- **Bills/Invoices:** Record purchase invoices with status tracking
- **Purchases:** Track individual purchase items with expiration dates

Invoice statuses are predefined in `db/database.sql:86-88`:
```sql
INSERT INTO estado (id, nombre) VALUES
    (1, 'Por pagar'),    -- Pending payment
    (2, 'Pagada');       -- Paid
```

### Sales Management

- **Sales:** Process sales transactions
- **Sales Details:** Track items sold in each transaction
- Each sale automatically updates inventory stock levels

### User Management

Three role levels control access:

<CodeGroup>
```text Super Admin (Role 1)
Full system access including:
- User management
- Audit logs
- All modules
```

```text Admin (Role 2)
Full access to operational modules:
- Inventory
- Sales
- Purchases
- Suppliers
(No audit access)
```

```text User (Role 3)
Limited operational access:
- View own profile
- No user management
- No audit logs
```
</CodeGroup>

The role structure is defined in `db/database.sql:159-162`:
```sql
INSERT INTO rol (id, nombre) VALUES
    (1, 'Super Admin'),
    (2, 'Admin'),
    (3, 'User');
```

### Audit Trail

All user actions are logged automatically in the `auditoria` table with:
- Timestamp
- User browser
- IP address
- URL accessed
- Action description
- User ID

Example from `views/login.php:89`:
```php
auditoria($_SERVER['REMOTE_ADDR'], $_SERVER['REQUEST_URI'], 
          'El usuario '.$username.' inició sesión', $userid);
```

## Navigation

The system uses friendly URLs through mod_rewrite (`.htaccess:2-40`):

- `/login` - Login page
- `/logout` - Logout
- `/articles` - Product inventory
- `/sales` - Sales module
- `/purchases` - Purchase module
- `/suppliers` - Supplier management
- `/laboratories` - Laboratory management
- `/presentations` - Presentation types
- `/bills` - Invoice management
- `/users` - User management (admin only)
- `/audit` - Audit logs (super admin only)

## Session Management

Sessions expire after 5 minutes of inactivity as configured in `views/login.php:76`:

```php
$_SESSION['start'] = time();
$_SESSION['expire'] = $_SESSION['start'] + (5 * 60);
```

Users can check "Remember me" during login to stay logged in for up to 365 days.

## Quick Troubleshooting

### Cannot Access Login Page

If `/login` shows a 404 error:

```bash
# Verify mod_rewrite is enabled
apache2ctl -M | grep rewrite

# Should show: rewrite_module (shared)
```

### Database Connection Failed

Check credentials in `db/db.php:6-7`:

```php
$_user = 'user';  // Update this
$_pass = 'pass';  // Update this
```

### Blank Page After Login

Clear browser cache and cookies, then try again. Sessions may conflict if you're testing with multiple credentials.
