## Overview

The laboratory management system maintains a catalog of pharmaceutical manufacturers and laboratories. Each laboratory represents a company that produces pharmaceutical products. This classification helps organize the product catalog and enables reporting by manufacturer.

## Database Schema

The `laboratorio` (laboratory) table stores manufacturer information:

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | int(11) | PRIMARY KEY, AUTO_INCREMENT | Unique laboratory identifier |
| `nombre` | varchar(45) | NULL allowed | Laboratory/manufacturer name |

<Note>
The laboratory table has a simple structure with only an ID and name field. This lightweight design makes it easy to manage a catalog of pharmaceutical manufacturers.
</Note>

## Deleting Laboratories

### Deletion Rules

Laboratories can only be deleted if they have no associated products:

```sql
SELECT COUNT(*) as total 
FROM articulo 
WHERE laboratorio_id = {id};
```

<Warning>
Laboratories with associated products cannot be deleted. This protects data integrity and prevents orphaned product records. If the count is greater than 0, deletion is blocked with the error: "Ese laboratorio esta relacionado con al menos un artículo, no puede ser eliminado."
</Warning>

### Delete Workflow

<Steps>
  <Step title="Initiate Deletion">
    Click the trash icon next to the laboratory
  </Step>
  
  <Step title="Confirm Action">
    Confirm the deletion in the alert dialog
  </Step>
  
  <Step title="System Check">
    The system checks for related products
  </Step>
  
  <Step title="Process Result">
    If no products exist, the laboratory is deleted. Otherwise, an error is displayed.
  </Step>
</Steps>

### Successful Deletion

When a laboratory has no dependencies:

```sql
DELETE FROM laboratorio WHERE id = {id};
```

The deletion is logged in the audit trail with user information.

## Laboratory Relationships

### Product Connection

Laboratories are connected to products through a foreign key:

```sql
-- In articulo table
CONSTRAINT `fk_articulo_laboratorio1` 
    FOREIGN KEY (`laboratorio_id`) REFERENCES `laboratorio` (`id`)
```

One laboratory can have multiple products:

```
Laboratory: Bayer
├── Product: Aspirin 500mg Tablets
├── Product: Ciprofloxacin 500mg Tablets
└── Product: Vitamin C 1000mg Tablets
```

### Product Organization

Products are organized by laboratory in several views:

**Product Creation Form:**
```sql
SELECT id as laboratorio_id, nombre as laboratorio_nombre 
FROM laboratorio 
WHERE 1;
```

**Product Listing:**
```sql
SELECT 
    laboratorio.nombre as laboratorio,
    articulo.nombre,
    ...
FROM articulo, laboratorio
WHERE laboratorio_id = laboratorio.id;
```

**Purchase Product Selection:**
Products are grouped by laboratory and presentation for easy selection during purchase entry.

## Data Entry Tips

### Naming Conventions

Maintain consistency in laboratory names:

**Correct:**
- ✅ "Bayer"
- ✅ "Pfizer Inc."
- ✅ "Novartis AG"

**Avoid:**
- ❌ "BAYER" (all caps)
- ❌ "bayer" (lowercase)
- ❌ "Bayer Lab" and "Bayer" (duplicates with variations)

### Character Limit

The name field is limited to 45 characters. For laboratories with long official names, use common abbreviations:

- ✅ "GlaxoSmithKline" or "GSK"
- ✅ "Bristol-Myers Squibb" or "BMS"

### Special Characters

The system handles special characters in laboratory names:

- Hyphens: "Bristol-Myers Squibb" ✅
- Parentheses: "Merck (MSD)" ✅
- Ampersands: "Johnson & Johnson" ✅

## Reporting and Analytics

### Laboratory Performance Reports

Use laboratory data for business intelligence:

**Top Performing Laboratories:**
```sql
SELECT 
    laboratorio.nombre,
    COUNT(articulo.id) as total_products,
    SUM(articulo.stock) as total_stock
FROM laboratorio
LEFT JOIN articulo ON laboratorio.id = articulo.laboratorio_id
GROUP BY laboratorio.id
ORDER BY total_products DESC;
```

**Sales by Laboratory:**
```sql
SELECT 
    laboratorio.nombre,
    SUM(venta_detalle.cantidad * articulo.valor_venta) as total_sales
FROM laboratorio
JOIN articulo ON laboratorio.id = articulo.laboratorio_id
JOIN venta_detalle ON articulo.id = venta_detalle.articulo_id
GROUP BY laboratorio.id
ORDER BY total_sales DESC;
```

Analyze how your inventory is distributed across laboratories to inform purchasing decisions and shelf space allocation.
