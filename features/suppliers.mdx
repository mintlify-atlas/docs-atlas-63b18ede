## Overview

The supplier management system maintains a comprehensive directory of pharmaceutical suppliers. Supplier records store essential business information used for purchase orders, invoicing, and communication. Each supplier must have a unique name and tax identification number (NIT).

## Database Schema

The `proveedor` (supplier) table stores all supplier information:

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | int(11) | PRIMARY KEY, AUTO_INCREMENT | Unique supplier identifier |
| `nombre` | varchar(45) | NOT NULL, UNIQUE | Supplier company name |
| `nit` | varchar(20) | NOT NULL | Tax identification number (NIT) |
| `direccion` | varchar(45) | NOT NULL | Physical business address |
| `telefono` | varchar(20) | NOT NULL | Contact phone number |
| `email` | varchar(45) | NOT NULL | Email address |

### Unique Constraint

```sql
UNIQUE KEY `nombre_UNIQUE` (`nombre`)
```

<Warning>
Supplier names must be unique in the system. Two suppliers cannot share the same company name, even if they have different NITs or addresses.
</Warning>

### Validation Rules

Before creating a supplier, the system validates uniqueness:

```sql
SELECT count(*) as total 
FROM proveedor 
WHERE nit = '{nit_proveedor}' 
   OR nombre = '{nombre_proveedor}';
```

<Warning>
If a supplier with the same NIT or name already exists, you'll receive an error: "Ese proveedor ya esta registrado"
</Warning>

### Insert Query

When validation passes:

```sql
INSERT INTO proveedor (
    nombre,
    nit,
    direccion,
    telefono,
    email
) VALUES (
    '{nombre_proveedor}',
    '{nit_proveedor}',
    '{direccion_proveedor}',
    '{telefono_proveedor}',
    '{email_proveedor}'
);
```

## Deleting Suppliers

### Deletion Rules

Suppliers can only be deleted if they have no associated invoices:

```sql
SELECT COUNT(*) as total 
FROM factura 
WHERE proveedor_id = {id};
```

<Warning>
Suppliers with invoice history cannot be deleted. This protects the integrity of purchase and financial records. If the count is greater than 0, deletion is blocked with the error: "Ese proveedor tiene facturas relacionadas, no puede ser eliminado."
</Warning>

### Delete Workflow

<Steps>
  <Step title="Initiate Deletion">
    Click the trash icon next to the supplier
  </Step>
  
  <Step title="Confirm Action">
    Confirm the deletion in the alert dialog
  </Step>
  
  <Step title="System Check">
    The system checks for related invoices
  </Step>
  
  <Step title="Process Result">
    If no invoices exist, the supplier is deleted. Otherwise, an error is displayed.
  </Step>
</Steps>

### Successful Deletion

When a supplier has no dependencies:

```sql
DELETE FROM proveedor WHERE id = {id};
```

The deletion is logged in the audit trail with user information.

## Supplier Relationships

### Invoice Connection

Suppliers are connected to the `factura` (invoice) table:

```sql
CONSTRAINT `fk_factura_proveedor` 
    FOREIGN KEY (`proveedor_id`) REFERENCES `proveedor` (`id`)
```

One supplier can have multiple invoices:
```
Supplier: ABC Pharmaceuticals
├── Invoice #1001 (2024-01-15)
├── Invoice #1002 (2024-02-03)
└── Invoice #1003 (2024-03-10)
```

### Purchase Traceability

Through invoices, suppliers are indirectly connected to purchases:

```
Supplier → Invoice (Factura) → Purchase (Compra) → Product (Articulo)
```

This relationship enables:
- Supplier performance analysis
- Purchase history by supplier
- Payment tracking
- Supplier comparison

## NIT (Tax Identification Number)

### What is a NIT?

NIT (Número de Identificación Tributaria) is a tax identification number used in many Latin American countries, particularly Colombia. It's similar to:

- EIN (Employer Identification Number) in the United States
- VAT number in European countries
- Tax ID in other jurisdictions

### Format and Validation

The system stores NIT as a varchar(20) field but does not enforce a specific format. Common NIT formats include:

- `123456789-0` (with verification digit)
- `123456789` (without hyphen)
- `123.456.789-0` (with dots and hyphen)

<Note>
Consider implementing format validation or standardization if your regulatory environment requires specific NIT formatting.
</Note>

## Address Management

### Physical Address

The `direccion` field stores the supplier's physical business address (up to 45 characters).

<Warning>
The 45-character limit may be restrictive for long addresses. Consider abbreviations or splitting into multiple address fields if needed.
</Warning>

### Address Best Practices

- Include street address and city
- Use standard abbreviations (St., Ave., Blvd.)
- Include postal/zip code if space permits
- Keep format consistent across all suppliers

## Data Entry Tips

### Standardization

Maintain consistency in data entry:

**Company Names:**
- ✅ "ABC Pharmaceuticals S.A."
- ❌ "abc pharma", "ABC Pharma SA"

**Phone Numbers:**
- ✅ "+57 300 123 4567"
- ✅ "(300) 123-4567"
- ❌ Mixed formats across suppliers

**Email Addresses:**
- ✅ "ventas@abcpharma.com"
- ✅ "compras@abcpharma.com"
- ❌ Personal email addresses

### Special Characters

The system sanitizes input to prevent:
- SQL injection
- HTML/script injection
- Single quotes (converted to `&#39;`)
