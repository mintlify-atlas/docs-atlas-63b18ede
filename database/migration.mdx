# Database Migration

This guide explains how to import and set up the Droguería Somos Salud database from the provided SQL file.

## Prerequisites

Before importing the database, ensure you have:

- MySQL 5.5.57 or higher installed
- MySQL root access or appropriate user privileges
- The `database.sql` file located in `/db/database.sql`

## Quick Start

### Using Command Line

The fastest way to import the database:

```bash
# Import the database
mysql -u root -p < /path/to/source/db/database.sql

# Verify import
mysql -u root -p -e "USE drogueria; SHOW TABLES;"
```

### Using MySQL Client

```bash
# Connect to MySQL
mysql -u root -p

# Run these commands:
mysql> source /path/to/source/db/database.sql;
mysql> USE drogueria;
mysql> SHOW TABLES;
mysql> quit;
```

## Step-by-Step Installation

### Step 1: Prepare MySQL

Connect to MySQL server:

```bash
mysql -u root -p
```

### Step 2: Create Database (Optional)

The SQL file automatically creates the database, but you can do it manually:

```sql
CREATE DATABASE IF NOT EXISTS drogueria 
    DEFAULT CHARACTER SET utf8 
    COLLATE utf8_swedish_ci;
```

Exit MySQL client and run:

```bash
mysql -u root -p drogueria < /path/to/source/db/database.sql
```

If you need to import specific tables:

```bash

mysql -u root -p drogueria < articulo.sql
```

## Database Character Set

The database uses UTF-8 encoding:

```sql
CREATE DATABASE IF NOT EXISTS `drogueria` 
    /*!40100 DEFAULT CHARACTER SET utf8 COLLATE utf8_swedish_ci */;
```

This ensures proper handling of Spanish characters and special symbols.

## Foreign Key Constraints

The import automatically creates foreign key relationships. The SQL file temporarily disables foreign key checks during import:

```sql
/*!40014 SET @OLD_FOREIGN_KEY_CHECKS=@@FOREIGN_KEY_CHECKS, FOREIGN_KEY_CHECKS=0 */;
-- ... table creation ...
/*!40014 SET FOREIGN_KEY_CHECKS=IF(@OLD_FOREIGN_KEY_CHECKS IS NULL, 1, @OLD_FOREIGN_KEY_CHECKS) */;
```

This allows tables to be created in any order without constraint violations.

## Backup Before Migration

If you're re-importing or updating an existing database:

### Create Backup

```bash
# Full database backup
mysqldump -u root -p drogueria > backup_drogueria_$(date +%Y%m%d).sql

# Backup with compression
mysqldump -u root -p drogueria | gzip > backup_drogueria_$(date +%Y%m%d).sql.gz
```

### Restore from Backup

```bash
# Restore uncompressed backup
mysql -u root -p drogueria < backup_drogueria_20260304.sql

# Restore compressed backup
gunzip < backup_drogueria_20260304.sql.gz | mysql -u root -p drogueria
```

## Post-Installation Tasks

### 1. Create Application User

For security, create a dedicated MySQL user for the application:

```sql
-- Create user
CREATE USER 'drogueria_user'@'localhost' IDENTIFIED BY 'secure_password';

-- Grant necessary privileges
GRANT SELECT, INSERT, UPDATE, DELETE ON drogueria.* TO 'drogueria_user'@'localhost';

-- Apply changes
FLUSH PRIVILEGES;
```

### 2. Update db.php Configuration

Update `/db/db.php` with the new credentials:

```php
$_user = 'drogueria_user';
$_pass = 'secure_password';
```

### 3. Verify Foreign Keys

Check that all foreign key constraints were created:

```sql
SELECT 
    TABLE_NAME,
    COLUMN_NAME,
    CONSTRAINT_NAME,
    REFERENCED_TABLE_NAME,
    REFERENCED_COLUMN_NAME
FROM
    INFORMATION_SCHEMA.KEY_COLUMN_USAGE
WHERE
    REFERENCED_TABLE_SCHEMA = 'drogueria'
    AND REFERENCED_TABLE_NAME IS NOT NULL;
```

### 4. Verify Indexes

Check that all indexes are in place:

```sql
SELECT 
    TABLE_NAME,
    INDEX_NAME,
    COLUMN_NAME
FROM
    INFORMATION_SCHEMA.STATISTICS
WHERE
    TABLE_SCHEMA = 'drogueria'
ORDER BY
    TABLE_NAME, INDEX_NAME;
```

## Troubleshooting

### Error: Database exists

```
ERROR 1007 (HY000): Can't create database 'drogueria'; database exists
```

**Solution**: Drop and recreate, or skip database creation:
```sql
DROP DATABASE IF EXISTS drogueria;
-- Then re-import
```

### Error: Access denied

```
ERROR 1045 (28000): Access denied for user 'root'@'localhost'
```

**Solution**: Verify MySQL credentials:
```bash
mysql -u root -p
# Enter correct password
```

### Error: Foreign key constraint fails

```
ERROR 1215 (HY000): Cannot add foreign key constraint
```

**Solution**: This shouldn't occur with the provided SQL file, but if it does:
- Ensure InnoDB engine is available
- Check that referenced tables exist
- Verify column types match between tables

### Error: Character encoding issues

```
ERROR: Unknown charset 'utf8'
```

**Solution**: Update MySQL configuration to support UTF-8:
```ini
[mysqld]
character-set-server=utf8
collation-server=utf8_swedish_ci
```
